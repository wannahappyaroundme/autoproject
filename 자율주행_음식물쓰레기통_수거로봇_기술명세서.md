## 4. 인식 (Perception)

### 4.1 쓰레기통 식별 전략: QR + Vision 하이브리드

프로젝트의 핵심 질문인 **"QR코드를 쓸 것인가, 비전만 쓸 것인가"** 에 대한 답은 **둘 다 사용하는 하이브리드 접근**입니다.

#### 4.1.1 QR코드 (1차 식별 — 신뢰성 보장)

| 항목 | 설명 |
|------|------|
| **역할** | 쓰레기통 고유 ID 부여 및 원거리 식별 |
| **라이브러리** | `pyzbar` 또는 OpenCV `QRCodeDetector` |
| **QR 내용** | JSON 형식: `{"bin_id": "A동-12-03", "type": "food_waste", "capacity": "3L"}` |
| **부착 위치** | 쓰레기통 전면 상단 (로봇 카메라 높이에 맞춤) |
| **인식 거리** | 10x10cm QR → 약 1.5~3m 거리에서 인식 가능 |
| **장점** | 100% 정확한 ID 매칭, 조명 영향 적음, 구현 간단 |
| **단점** | QR이 오염/훼손될 수 있음, 부착 필요 |

```python
# QR 인식 노드 핵심 로직 (예시)
import cv2
from pyzbar.pyzbar import decode

def qr_callback(self, img_msg):
    frame = self.bridge.imgmsg_to_cv2(img_msg, "bgr8")
    qr_codes = decode(frame)
    for qr in qr_codes:
        data = json.loads(qr.data.decode('utf-8'))
        bin_id = data['bin_id']
        # QR의 4꼭짓점으로 상대 위치 추정 (PnP)
        points = np.array(qr.polygon, dtype=np.float32)
        _, rvec, tvec = cv2.solvePnP(obj_pts, points, K, dist)
        self.publish_bin_pose(bin_id, tvec)
```

#### 4.1.2 딥러닝 객체 탐지 (2차 식별 — 정밀 위치 추정)

| 항목 | 설명 |
|------|------|
| **모델** | **YOLOv11n (Nano)** — SOTA 실시간 객체 탐지 |
| **선택 이유** | YOLOv11은 C3k2 블록, 공간 어텐션(C2PSA) 등으로 파라미터 대비 최고 효율 달성. Nano 버전은 Jetson Nano에서 TensorRT 가속 시 ~30 FPS 가능 |
| **대안 모델** | YOLOv8n (더 안정적인 생태계), YOLOv10n (NMS-free로 레이턴시 46% 감소) |
| **입력** | 640×640 RGB 이미지 |
| **출력** | Bounding Box + Class (food_waste_bin, obstacle, person 등) + Confidence |
| **커스텀 학습 데이터** | 아파트 음식물쓰레기통 이미지 500~1000장 (Roboflow로 어노테이션) |
| **추론 엔진** | NVIDIA TensorRT (FP16 양자화) |
| **프레임워크** | Ultralytics + PyTorch → TensorRT export |

**YOLOv11n 성능 벤치마크 (COCO)**:
- mAP@0.5: 39.5%
- 파라미터: 2.6M
- FLOPs: 6.5G
- Jetson Nano TensorRT FP16 추론: ~25-35ms/frame

```python
# YOLOv11n 학습 명령어
from ultralytics import YOLO

model = YOLO('yolo11n.pt')  # pretrained nano 모델
model.train(
    data='food_waste_bin.yaml',  # 커스텀 데이터셋
    epochs=100,
    imgsz=640,
    batch=16,
    device=0  # GPU
)

# TensorRT export (Jetson Nano에 배포)
model.export(format='engine', half=True)  # FP16
```

**커스텀 데이터셋 구성 (`food_waste_bin.yaml`)**:
```yaml
path: ./datasets/food_waste_bin
train: images/train
val: images/val
test: images/test

names:
  0: food_waste_bin    # 음식물쓰레기통
  1: general_bin       # 일반쓰레기통 (혼동 방지)
  2: person            # 사람 (안전)
  3: vehicle           # 차량
  4: qr_marker         # QR마커 영역
```

#### 4.1.3 하이브리드 인식 파이프라인

```
카메라 프레임 입력
    │
    ├── [파이프라인 A: QR 인식] ──→ bin_id + 대략적 위치
    │       pyzbar (CPU, ~5ms)
    │
    └── [파이프라인 B: YOLO 객체탐지] ──→ bounding box + class
            YOLOv11n-TensorRT (GPU, ~30ms)
            │
            └── [Depth 정합] ──→ 3D 위치 (x,y,z)
                 RealSense depth + bbox center
    │
    ▼
[퓨전 노드] ──→ 최종 쓰레기통 Pose (position + orientation)
    │
    - QR 인식 성공 → QR 기반 ID + YOLO 기반 정밀 위치
    - QR 인식 실패 → YOLO만으로 클래스 식별 + depth 위치
    - 둘 다 실패 → 탐색 행동 (회전/전진) 트리거
```

### 4.2 Depth 기반 3D 위치 추정

RGB-D 카메라(RealSense D435)의 depth 데이터를 활용하여 2D 바운딩 박스를 3D 좌표로 변환합니다.

```python
# YOLO bbox 중심점의 3D 좌표 추출
def get_3d_position(bbox, depth_image, camera_intrinsics):
    cx, cy = (bbox[0] + bbox[2]) / 2, (bbox[1] + bbox[3]) / 2
    depth = depth_image[int(cy), int(cx)]  # mm 단위
    
    # 카메라 내부 파라미터로 역투영
    fx, fy = camera_intrinsics.fx, camera_intrinsics.fy
    ppx, ppy = camera_intrinsics.ppx, camera_intrinsics.ppy
    
    x = (cx - ppx) * depth / fx
    y = (cy - ppy) * depth / fy
    z = depth
    
    return np.array([x, y, z]) / 1000.0  # meter 변환
```

## 5. 위치추정 및 지도작성 (SLAM)

### 5.1 SLAM 알고리즘 선택

| 알고리즘 | 센서 | Jetson Nano 호환 | 정확도 | 학부 난이도 | 선택 |
|---------|------|-----------------|-------|-----------|------|
| **RTAB-Map** | RGB-D + (LiDAR) | ✅ (최적화 필요) | ★★★★ | ★★★ | **✅ 1순위** |
| **Cartographer** | LiDAR + IMU | ✅ | ★★★★★ | ★★★★ | LiDAR 사용 시 |
| **ORB-SLAM3** | 모노/스테레오/RGB-D | ⚠️ (메모리 부족 가능) | ★★★★★ | ★★★★★ | 중급 업그레이드 |
| **GMapping** | LiDAR | ✅ | ★★★ | ★★ | 가장 쉬움 |
| **SLAM Toolbox** | LiDAR | ✅ | ★★★★ | ★★★ | Nav2 기본 연동 |

### 5.2 학부 수준 권장: RTAB-Map

**선택 이유**:
- RGB-D 카메라만으로 SLAM 가능 (LiDAR 없이도 작동)
- 2D 점유격자 지도(Occupancy Grid) + 3D 포인트클라우드 동시 생성
- ROS 2 Nav2와 자연스럽게 연동
- Loop Closure 자동 감지
- Jetson Nano에서 실제 구동 사례 다수 존재

```bash
# RTAB-Map 실행 (RealSense D435)
ros2 launch rtabmap_launch rtabmap.launch.py \
    rgb_topic:=/camera/color/image_raw \
    depth_topic:=/camera/aligned_depth_to_color/image_raw \
    camera_info_topic:=/camera/color/camera_info \
    frame_id:=base_link \
    approx_sync:=true \
    qos:=2
```

**RTAB-Map 주요 파라미터 튜닝**:
```yaml
rtabmap:
  ros__parameters:
    Mem/IncrementalMemory: "true"
    Mem/InitWMWithAllNodes: "false"
    RGBD/NeighborLinkRefining: "true"
    RGBD/ProximityBySpace: "true"
    RGBD/OptimizeFromGraphEnd: "false"
    Reg/Strategy: "0"              # 0=Visual, 1=ICP, 2=Visual+ICP
    Vis/MinInliers: "15"           # 최소 매칭 특징점 수
    Grid/CellSize: "0.05"          # 5cm 해상도 격자 지도
    Grid/RangeMax: "5.0"           # 최대 5m 범위 매핑
    Grid/FootprintHeight: "0.4"    # 로봇 높이
```

### 5.3 센서 퓨전 (robot_localization)

휠 인코더 오도메트리 + IMU + Visual Odometry를 EKF(Extended Kalman Filter)로 융합합니다.

```yaml
# ekf.yaml — robot_localization 설정
ekf_node:
  ros__parameters:
    frequency: 30.0
    sensor_timeout: 0.1
    two_d_mode: true           # 2D 평면 주행이므로
    
    odom0: /wheel_odom         # 휠 인코더
    odom0_config: [true, true, false,   # x, y, z
                   false, false, true,   # roll, pitch, yaw
                   true, true, false,    # vx, vy, vz
                   false, false, true,   # vroll, vpitch, vyaw
                   false, false, false]  # ax, ay, az
    
    imu0: /imu/data            # MPU-9250
    imu0_config: [false, false, false,
                  false, false, true,    # yaw만 사용
                  false, false, false,
                  false, false, true,    # vyaw
                  true, false, false]    # ax (선형가속도)
```

## 6. 자율주행 내비게이션 (Navigation)

### 6.1 Nav2 (Navigation2) 프레임워크

Nav2는 ROS 2의 산업 표준 자율주행 프레임워크로, 100개 이상의 기업에서 프로덕션에 사용합니다.

### 6.2 Nav2 핵심 컴포넌트

```
┌────────────────────────────────────────────┐
│           BT Navigator                      │
│  (Behavior Tree 기반 미션 스케줄링)          │
│                                              │
```

### 7.1 다층 장애물 감지 전략

```
레벨 1: 정적 장애물 (지도 기반)
  └── SLAM으로 생성된 점유격자 맵의 장애물 정보

레벨 2: 동적 장애물 (센서 실시간)
  └── RGB-D 포인트클라우드 + (LiDAR 스캔)
      → Local Costmap Obstacle Layer
      → DWB/RPP가 실시간 회피 경로 계산

레벨 3: 긴급 정지 (Safety)
  └── 근접 센서 또는 depth 최소 거리 모니터링
      → 0.3m 이내 장애물 감지 시 즉시 정지
      → /cmd_vel에 zero velocity 오버라이드
```

### 7.2 사람 감지 및 안전 모드

YOLOv11n에서 "person" 클래스 감지 시 특별 행동을 트리거합니다.

```python
# safety_monitor_node.py
class SafetyMonitor(Node):
    def detection_callback(self, msg):
        for det in msg.detections:
            if det.class_name == "person":
                distance = self.get_depth_at_bbox(det.bbox)
                if distance < 1.0:  # 1m 이내
                    self.publish_stop_command()
                    self.set_speed_limit(0.0)  # 즉시 정지
                elif distance < 2.0:  # 2m 이내
                    self.set_speed_limit(0.2)  # 감속
                elif distance < 3.0:
                    self.set_speed_limit(0.5)  # 서행
```

### 7.3 Recovery Behaviors

Nav2의 기본 복구 행동을 활용합니다:

- **Spin**: 제자리 회전으로 새로운 경로 탐색
- **BackUp**: 후진으로 막힌 상황 탈출
- **Wait**: 동적 장애물(사람, 차량) 통과 대기
- **Clear Costmap**: 센서 노이즈로 인한 가짜 장애물 제거

## 8. 쓰레기통 탐지 및 파지 제어

### 8.1 파지 시퀀스 상세

```
[대기] → [탐지] → [접근] → [정렬] → [파지] → [확인] → [이동]

1. 탐지 (Detection)
   - QR 스캔으로 목표 쓰레기통 ID 확인
   - YOLOv11n으로 바운딩 박스 + depth → 3D 위치
   
2. 접근 (Approach)
   - 쓰레기통 정면 0.5m 지점까지 Nav2로 이동
   - Visual Servoing으로 정밀 정렬 (카메라 피드백 제어)
   
3. 정렬 (Alignment)
   - 쓰레기통 중심과 로봇 E/ㄷ형 그리퍼 중심 정렬
   - 오차 허용: ±3cm (lateral), ±5° (yaw)
   
4. 파지 (Grasp)
   - 그리퍼 열림 → 전진 15cm → 그리퍼 닫힘
   - 서보모터 토크 피드백으로 파지 성공 확인
   
5. 확인 (Verification)
   - 무게 센서 또는 전류 모니터링으로 파지 확인
   - 실패 시 재시도 (최대 3회)
```

### 8.2 Visual Servoing (정밀 접근)

Nav2 내비게이션이 대략적 위치까지 이동한 후, 카메라 피드백 기반 제어로 정밀 접근합니다.

```python
# visual_servoing_node.py — P제어 기반 간단한 구현
class VisualServoingNode(Node):
    def __init__(self):
        super().__init__('visual_servoing')
        self.Kp_linear = 0.5   # 전진 게인
        self.Kp_angular = 1.0  # 회전 게인
        self.target_distance = 0.15  # 목표 거리 (파지 직전)
    
    def servo_callback(self, bin_pose_msg):
        # 카메라 좌표계에서의 쓰레기통 위치
        dx = bin_pose_msg.pose.position.z  # depth = 전방 거리
        dy = bin_pose_msg.pose.position.x  # 좌우 오프셋
        
        cmd = Twist()
        
        if dx > self.target_distance:
            cmd.linear.x = self.Kp_linear * (dx - self.target_distance)
            cmd.linear.x = min(cmd.linear.x, 0.15)  # 속도 제한
        
        cmd.angular.z = -self.Kp_angular * dy  # 좌우 정렬
        cmd.angular.z = np.clip(cmd.angular.z, -0.3, 0.3)
        
        self.cmd_pub.publish(cmd)
```

```python
# gripper_controller.py
class GripperController(Node):
    """E/ㄷ형 그리퍼 — 서보모터 2개 (좌/우 암)"""
    
    OPEN_ANGLE = 90    # 열림 각도
    CLOSE_ANGLE = 30   # 닫힘 각도 (쓰레기통 폭에 맞춤)
    
    def open_gripper(self):
        self.set_servo(self.LEFT_SERVO, self.OPEN_ANGLE)
        self.set_servo(self.RIGHT_SERVO, self.OPEN_ANGLE)
    
    def close_gripper(self):
        self.set_servo(self.LEFT_SERVO, self.CLOSE_ANGLE)
        self.set_servo(self.RIGHT_SERVO, self.CLOSE_ANGLE)
        time.sleep(0.5)
        
        # 토크 피드백 확인 (전류 측정)
        current = self.read_servo_current()
        if current > self.GRASP_THRESHOLD:
            return True   # 파지 성공
        return False      # 파지 실패
    
    def set_servo(self, servo_id, angle):
        # Arduino/ESP32로 시리얼 명령 전송
        msg = f"SERVO,{servo_id},{angle}\n"
        self.serial_port.write(msg.encode())
```

```xml
          <mass>2.5</mass>  <!-- 3L 빈통 + 내용물 -->
        </inertial>
      </link>
    </model>
    
    <!-- 장애물: 주차 차량, 화분, 볼라드 등 -->
    <include><uri>model://parked_car</uri></include>
    <include><uri>model://bollard</uri></include>
  </world>
</sdf>
```

### 11.4 중급 시뮬레이터: NVIDIA Isaac Sim

- 포토리얼리스틱 렌더링 (합성 데이터 생성에 유리)
- GPU 기반 물리 시뮬레이션
- Domain Randomization으로 Sim-to-Real 갭 축소
- Isaac ROS와 직접 연동
- **단점**: 최소 RTX 3070 이상 GPU 필요, 학습 곡선 높음

## 12. 학부 수준 vs 중급 업그레이드 비교표

### 12.1 소프트웨어 스택 비교

| 카테고리 | 학부 수준 (60~80만원) | 중급 업그레이드 | 변화 요약 |
|---------|---------------------|---------------|----------|
| **컴퓨터** | Jetson Nano 4GB | Jetson Orin Nano 8GB | GPU 성능 ~80배, 메모리 2배 |
| **OS** | Ubuntu 20.04 + JetPack 4.6 | Ubuntu 22.04 + JetPack 6.x | ROS 2 Humble 네이티브 |
| **객체탐지** | YOLOv11n (TensorRT FP16) | YOLOv11s/m (TensorRT INT8) | mAP +5~10%, 더 큰 모델 가능 |
| **SLAM** | RTAB-Map (RGB-D) | ORB-SLAM3 (Visual-Inertial) + RTAB-Map | 더 정확한 위치추정, 멀티맵 |
| **LiDAR** | 없음 (RGB-D만) | RPLiDAR A1 추가 | 360° 장애물 감지, Cartographer 사용 가능 |
| **Global Planner** | NavFn (A*) | Smac Hybrid-A* | 동역학 고려 경로, 더 부드러운 주행 |
| **Local Planner** | DWB | MPPI Controller | 예측 기반 최적 제어, 동적 장애물 우수 |
| **센서 퓨전** | EKF (wheel + IMU) | UKF (wheel + IMU + Visual Odom) | 비선형 모델 더 정확 |
| **시뮬레이션** | Gazebo Classic | Isaac Sim + Gazebo | 포토리얼리스틱, Domain Randomization |
| **관제 웹** | Flask + SQLite | FastAPI + PostgreSQL + Redis | 확장성, 실시간 처리 성능 향상 |
| **통신** | MQTT + Wi-Fi | MQTT + Wi-Fi + ROS 2 DDS Bridge | 멀티로봇 지원, QoS 보장 |

### 14.1 반드시 사전 정의해야 할 항목

## 부록 B: 개발 환경 빠른 세팅 가이드

```bash
# === 1. Jetson Nano 기본 설정 ===
# JetPack 4.6 SD 이미지 플래싱 후:
sudo apt update && sudo apt upgrade -y

# Swap 메모리 확장 (ORB-SLAM3 등 메모리 부족 대비)
sudo fallocate -l 8G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# === 2. ROS 2 Humble 설치 ===
# (Jetson Nano는 ARM64 — 공식 바이너리 또는 소스 빌드)
# Docker 방식 추천:
docker pull dustynv/ros:humble-ros-base-l4t-r32.7.1

# === 3. Python 환경 ===
pip3 install ultralytics pyzbar paho-mqtt fastapi uvicorn

# === 4. 개발 PC에서 시뮬레이션 (x86) ===
sudo apt install ros-humble-desktop
sudo apt install ros-humble-navigation2
sudo apt install ros-humble-turtlebot3-gazebo
```
