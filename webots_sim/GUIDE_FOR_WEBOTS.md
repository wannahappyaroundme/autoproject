# Webots 담당자 가이드 (비개발자용)

코드 실행이나 프로그래밍 지식은 전혀 필요 없습니다.
Webots에서 3D 환경만 수정하면 됩니다.

---

## 1단계: 프로그램 설치 (최초 1회)

### Webots 설치

1. https://cyberbotics.com 접속
2. **Download** 클릭
3. 본인 OS에 맞는 버전 다운로드 (Mac / Windows / Linux)
4. 설치 후 실행해서 정상 동작 확인

### GitHub Desktop 설치 (파일 주고받기용)

명령어 대신 **클릭**으로 GitHub 사용할 수 있는 프로그램입니다.

1. https://desktop.github.com 접속
2. 다운로드 + 설치
3. 실행 → **Sign in to GitHub.com** 클릭 → GitHub 계정으로 로그인
4. 아직 GitHub 계정이 없으면 https://github.com/join 에서 무료 가입

---

## 2단계: 프로젝트 가져오기 (최초 1회)

1. GitHub Desktop 실행
2. 상단 메뉴 → **File** → **Clone Repository**
3. **URL** 탭 클릭
4. 아래 주소 입력:
   ```
   https://github.com/wannahappyaroundme/autoproject.git
   ```
5. **Local Path**: 원하는 폴더 선택 (예: 바탕화면)
6. **Clone** 클릭 → 다운로드 완료

---

## 3단계: Webots에서 월드 열기

1. Webots 실행
2. **File** → **Open World**
3. 아까 다운받은 폴더로 이동:
   ```
   autoproject → webots_sim → worlds → apartment_complex.wbt
   ```
4. 열기 → 아파트 단지 3D 환경이 나타남

처음 열면 이렇게 보입니다:
- 아파트 건물 20개 (101~106동, 동마다 색상이 다름)
- 건물 사이 도로 (회색 바닥)
- 흰색 작은 쓰레기통 16개 (뚜껑이 열리는 Trashcan)
- 노란색 바닥 + 지붕 = 집하장 (로봇 출발/복귀 지점)
- 수거 로봇 4대 (Robot_1은 키보드 조작 가능, Robot_2~4는 ROS 2 연동용)
- 녹색 블록 = 놀이터, 주황색 = 경비실, 회색 = 관리사무소/주차장

---

## 4단계: 수정하기

### Webots에서 수정하는 방법

왼쪽 **Scene Tree** 패널에서 원하는 물체를 클릭하면 속성을 변경할 수 있습니다.

### 수정해도 되는 것

| 항목 | 수정 방법 | 예시 |
|------|----------|------|
| 건물 위치 | Scene Tree에서 건물 클릭 → translation 값 변경 | 101동을 옆으로 이동 |
| 건물 크기 | size 값 변경 | 건물을 더 크게/작게 |
| 건물 색상 | baseColor 값 변경 | 회색 → 베이지 |
| 도로 | 바닥 텍스처/색상 변경 | 아스팔트 색 조정 |
| 쓰레기통 위치 | 쓰레기통 클릭 → translation 변경 | 쓰레기통을 다른 입구로 이동 |
| 쓰레기통 모양 | 크기, 색상 변경 | |
| 놀이터/주차장 | 위치, 크기, 색상 변경 | |
| 조명 | Sun 밝기/각도 변경 | 낮/저녁 분위기 |
| 나무/벤치 추가 | Scene Tree에서 우클릭 → Add New | 환경 꾸미기 |

### 절대 수정하면 안 되는 것

⚠️ 아래를 바꾸면 로봇 소프트웨어가 고장납니다!

| 건드리지 마세요 | 이유 |
|----------------|------|
| Robot_1의 **controller** 항목 | `Robot_controller` 유지 (키보드 테스트용) |
| Robot_2~4의 **controller** 항목 | `<extern>` 유지해야 ROS 2 소프트웨어 연결됨 |
| 로봇의 **name** | `Robot_1`~`Robot_4` 유지 |
| 로봇 내부 센서/모터 이름 | 소프트웨어가 이 이름으로 연결함 |

간단하게: **로봇은 건드리지 말고, 주변 환경(건물/도로/쓰레기통)만 수정하세요.**

### Robot_1 키보드 조작 방법
시뮬레이션 실행(▶) 후 3D 뷰를 클릭하고:
- **위쪽 화살표**: 전진
- **아래쪽 화살표**: 후진
- **왼쪽 화살표**: 좌회전
- **오른쪽 화살표**: 우회전

## 5단계: 저장 + 업로드

### 저장

Webots에서 **File** → **Save World** (Ctrl+S / Cmd+S)

### GitHub에 올리기 (3번 클릭)

1. **GitHub Desktop** 열기
2. 왼쪽에 변경된 파일 목록이 보임 (체크 표시 확인)
3. 하단에 **Summary** 칸에 뭘 바꿨는지 적기
   - 예: `건물 위치 조정`, `쓰레기통 추가`
4. **Commit to main** 클릭 (파란 버튼)
5. 상단 **Push origin** 클릭

끝! PM이 pull하면 변경 사항을 받을 수 있습니다.

### PM이 올린 변경사항 받기

1. GitHub Desktop 열기
2. 상단 **Fetch origin** 클릭
3. **Pull origin** 클릭 (있으면)

---

## 6단계: 쓰레기통 위치 변경 시 PM에게 알려주기

쓰레기통을 옮기거나 추가/삭제하면, PM에게 **좌표를 알려줘야** 합니다.
(소프트웨어에서 경로를 계산할 때 이 좌표를 사용하기 때문)

### 좌표 확인 방법

1. Webots에서 쓰레기통 클릭
2. 왼쪽 Scene Tree에서 **translation** 값 확인
   - 예: `translation 7 0.2 8`
3. **X값**과 **Z값**만 알려주면 됩니다
   - X = 7, Z = 8 → PM에게 **(7, 8)** 이라고 알려주기

### 예시 메시지

> "101동 앞 쓰레기통 (7,8)에서 (9,8)로 옮겼어요.
> 공원 쪽에 새 쓰레기통 (42,16) 추가했어요."

## 문제가 생겼을 때

| 문제 | 해결 |
|------|------|
| 월드 파일이 안 열림 | Webots 버전 **R2025a** 이상인지 확인 |
| 로봇이 안 보임 | Scene Tree에서 WasteRobot 찾아서 클릭 → 카메라가 로봇으로 이동 |
| 저장이 안 됨 | File → Save World As로 다른 이름으로 저장 시도 |
| GitHub Desktop에서 에러 | Repository → Pull 먼저 해보기 (충돌 해결) |
| 실수로 로봇을 건드림 | Ctrl+Z로 되돌리기. 안 되면 GitHub Desktop에서 해당 파일 우클릭 → Discard Changes |
